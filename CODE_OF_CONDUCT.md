# Code of Conduct

TranslateR contributors are expected to keep project discussion respectful,
practical, and focused on improving the software.

## Expected Behavior

- Be respectful of translators, maintainers, and contributors.
- Assume good intent, but be clear when behavior or code needs improvement.
- Keep feedback specific and actionable.
- Respect the linguistic and cultural diversity of translation work.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or behavior.
- Dismissing language/script needs as unimportant.
- Publishing private information without permission.

## Reporting

If this project later has public maintainers, reports should go to the listed
maintainer contact. Until then, open an issue with enough detail for maintainers
to understand and address the problem.

## Scope

This code of conduct applies to project spaces such as issues, pull requests,
documentation, discussions, and related community channels.
